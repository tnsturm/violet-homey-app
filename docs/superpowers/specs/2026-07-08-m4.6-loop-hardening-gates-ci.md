## §3 Commit-Test-Gate — Entscheidungen

- **D1 — separat, nicht kombiniert:** eigener Hook `test-gate.js` NEBEN `typecheck-gate.js`
  (beide PreToolUse/Bash). Begründung: Claude Code führt matchende Hooks **parallel** aus →
  Commit-Latenz = max(tsc, npm test) statt Summe; Ein-Zweck-Hooks bleiben einzeln fail-open,
  testbar und M4.9-extrahierbar (Muster `check-version-sync.js`). Entscheidung wird zusätzlich
  im Dashboard-Log dokumentiert (Prompt-Auflage).
- **D2 — Testkommando aus `package.json` `scripts.test`** lesen (nicht `npm test` hardcoden):
  kein npm-Startup-Overhead/`npm.cmd`-Sonderfall auf Windows, und das Kommando bleibt für die
  M4.9-Framework-Extraktion eine Variable. Ausführung `spawnSync(script, {shell: true, cwd})`.
  Kein `scripts.test` → exit 0 (nicht unser Repo, fail-open).
- **D3 — Trigger-Regex** wie typecheck-gate: `/\bgit\s+commit\b/`.
- **D4 — Rekursion:** begrenzt by design — der Hook läuft nur, wenn Claude `git commit`
  aufruft; die Smoke-Tests spawnen den Hook direkt mit Fixture-cwd (Tiefe ≤ 2, Mini-Fixtures).
  Kein Env-Guard nötig (§2 Simplicity).
- **Latenz-Budget:** volle Suite ~5–8 s pro Commit, parallel zum tsc-Gate (~2 s) → ≈ heutige
  Suite-Laufzeit; M4.5-Cap (2×B_test = 8,86 s) bleibt der Wächter.

## §4 Release-Gate — Entscheidungen

- **Trigger:** `/\b(?:npx\s+)?homey\s+app\s+(install|publish)\b/` (PreToolUse/Bash).
- **Versionsquelle:** `.homeycompose/app.json` (Source of Truth, HOMEY.md). Nicht lesbar/kein
  Compose-Repo → exit 0 (nicht unser Repo, fail-open).
- **Bedingungen (alle Verstöße gesammelt melden, dann exit 2):**
  - (a) `.homeychangelog.json` hat für die Version **beide** nicht-leeren Einträge `en` UND
    `de`. Fehlende/unparsbare Datei = Verstoß (das IST der fehlende Changelog), Meldung
    verweist auf die HOMEY.md-JSON-Authoring-Regel.
  - (b) Version darf noch **nicht** in `docs/dashboard/versions.md` stehen (Suche nach
    `` `X.Y.Z` ``, Backtick-getreu zur Log-Tabelle). Datei fehlt → Check übersprungen.
  - (c) nur bei **publish**: `docs/superpowers/security/credential-rotation.md` existiert und
    enthält ein Datum `YYYY-MM-DD` (Rotations-Nachweis, Memory
    `security-rotate-violet-credential`). Sonst Verstoß.
- install prüft (a)+(b), publish prüft (a)+(b)+(c).

## §5 GitHub-Actions-CI — Entscheidungen

- **Datei:** `.github/workflows/ci.yml`, ein Job (`ubuntu-latest`, Node 18 — Homey-Pro-Runtime).
- **Trigger:** `pull_request`, `push` auf `main`, `schedule` (nightly Cron, läuft auf dem
  Default-Branch = main), `workflow_dispatch` (manueller Re-Run).
- **Steps:** checkout → setup-node (npm-Cache) → `npm ci` → `npm test` → `npm run typecheck`
  → Manifest-Sync (Node-Einzeiler: `app.json`-Version === `.homeycompose/app.json`-Version,
  Spiegel des `check-version-sync`-Hooks) → `npx --yes homey app validate --level publish`
  (per B2 ohne Login möglich). Fallback, falls npx-CLI auf dem Runner unerwartet hakt:
  offizielle Action `athombv/github-action-homey-app-validate` (Befund im Log festhalten).
- **Done:** erster grüner Lauf auf origin/main (Prompt-/§6-Kriterium).
